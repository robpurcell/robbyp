Ratpack and Lazybones

Aug 4th, 2013

So previously I started trying to figure out VertX. While I still like the idea of that, I’ve detoured into looking at Ratpack as it appeals to the old Grails hand in me. While I’m fond of Grails, and looking forward to seeing where it goes, I can’t help but feel that the lightweight approach is what I need to start our in my adventure of building a HTML 5 app with AngularJS and Bootstrap. See here for my tentative first steps with the client-side. Hopefully this will all come together.

Used Lazybones to create a template project, having setup the Github repo

lazybones create ratpack .

It sets up a nice, basic project layout.

I needed to add the IDEA plugin to build.gradle

./gradlew idea

Vert.x With a Gradle Build

Jul 24th, 2013

I previously set up Vert.x with a Gradle build. Now it’s time to have a play with it and see what we can do.

$ ./gradlew test

A lot of stuff downloads!

./gradlew idea

To install the support for IntelliJ.

Moving on to the dev guide.

Downloaded the examples. Running the Groovy webapp:

cd vertx-examples/src/modules/groovy/webapp
vertx runmod io.vertx~example-web-app~1.0

As webapp has a mods folder, the vertx command can discover the module and run it. See here for details.
